# Incident Detector / Emergent Quest Logic

## Principle

Quests are detected from world-state, not hand-issued as static missions.

## Incident structure

- id/title
- visibleProblem
- hiddenCause
- involvedAgentIds
- knownFacts
- possibleResolutions
- status
- resolutionState

## The Missing Delivery trigger

```txt
delivery_failed
+ Sara stock low
+ rumor connecting Sara to Registry
+ Malik suspicion toward Sara >= 50
→ create incident
```

## Resolution paths

- peaceful mediation
- investigation/evidence
- payment/alternate delivery
- counter-rumor
- Registry report
- founder delivery workflow
- exploit/chaos path
