# Visual Gameplay Shell v1

Status: **implemented on `feature/visual-gameplay-shell-v1`**

## Goal

Upgrade the static/live web play UI from a two-column debug panel into a **playable visual game shell** — location scenes, hotspot cards, NPC interaction cards, case board tabs, rumor trail, Leno overlay, founder panel, and branch-before-decision modal — without React or new npm dependencies.

## Layout

```
Topbar:     world · day · time · money · reputation · energy · Leno · branch
Left:       district SVG map + current location scene image
Center:     hotspot rail + dialogue + consequence panels
Right:      NPC cards · Leno · case board tabs · rumor trail · founder · major decisions
Bottom:     event feed · live consequence ticker · command fallback (buttons + text input)
```

Generated by `npm run play:web` → `static-play/index.html`.

## Modules

| Module | Role |
|--------|------|
| `src/play/visual-game-shell.js` | Game-layout HTML fragments (topbar, hotspots, NPC cards, case board tabs, modal) |
| `src/play/game-shell-model.js` | View-model: hotspots, NPC actions, case board, rumor trail, founder, major decisions |
| `src/play/web-renderer.js` | Assembles full page; re-exports `escapeHtml` / `applyLenoGuard` via `html-utils.js` |
| `src/play/html-utils.js` | Shared escape + Leno evidence guard |
| `src/cli/play-web.js` | `APP_CSS` game skin + live `APP_JS` (modal, tabs, ticker, `/api/command` updates) |
| `content/worldmind/content-pack-v1.json` | Authored hotspots with `description`, `possibleEvidence`, optional `icon` |

## Live play (`npm run play:server`)

`/api/command` responses update without reload:

- Dialogue (`#wm-dialogue`)
- Consequence panel + `[data-consequence-ticker]`
- Topbar money / reputation / energy
- Case board + rumor trail when `gameShell` returned
- Founder stats when `founder` returned
- Event feed prepend

**Branch-before-decision:** client matches `gameShell.majorDecisions` embedded in `#wm-state` before dispatch; modal offers save+branch, continue without branch, or cancel.

### Expected runtime fields (Leno `feature/runtime-gameplay-core-v1`)

| Field | UI use | Fallback |
|-------|--------|----------|
| `result.gameShell` | Full panel refresh | Rebuild from `playerKnowledge` client-side N/A — static shell only |
| `result.playerSnapshot` | Topbar money/rep/energy | `world.agents.player.stats` |
| `result.founder` | Founder panel | `world.founder` summary |
| `result.consequence.*Delta` | Ticker + topbar | Show zeros / omit |
| `result.majorDecisionPrompt` | Banner only (pre-check preferred) | Client `matchMajorDecision` |

## Security

- No `hiddenCause` or agent `secrets` in embedded state or rendered HTML.
- Leno panel uses `applyLenoGuard` — source-defining Nadia mentions redacted until `rumor_source_nadia` evidence.
- Case board suspect Nadia row redacted until same evidence.
- `rumor_source_nadia` link row redacted until evidence.

## Truly playable vs UI foundation

| Playable now | UI foundation / needs runtime branch |
|--------------|--------------------------------------|
| Location scene + hotspot inspect/run buttons | Hotspot-specific outcomes beyond generic `inspect` |
| NPC talk/ask/pay buttons | Topic-specific dialogue trees from runtime |
| Case board inspect/trace/counter buttons | Counter-rumor backfire simulation depth |
| Rumor trail display + actions | Full trace chain state from runtime |
| Founder panel buttons (when unlocked) | Delivery workflow state machine (runtime) |
| Major-decision modal + branch API | Auto-unlock founder from incident resolution |
| Live command execution via play-server | All consequence deltas if runtime omits fields |

## Branding frame (worldmind-site)

Production play portals wrap content in **`PlayBrandFrame`** — full-viewport vignette + Nordic cyberpunk border (`play-brand-frame` CSS class, aligned with static-dashboard depth). **`PlayBrandedHeader`** renders `gameShell.topbar` (world, day, time, money, branch) plus core engine version from `/api/health`.

Used on `/play` (2D portal) and `/play/3d`. The core static shell (`static-play/index.html` via `play-web.js`) uses a similar game skin in `APP_CSS` but does not mount the React frame components.

## Case board asset icons

`buildGameplayShellModel()` attaches **`gameShell.assets`** — stable UI icon paths from `WORLD_ASSETS.ui` in `src/play/assets.js`:

| Field | Path | UI use |
|-------|------|--------|
| `evidenceIcon` | `assets/ui/evidence-card.png` | Evidence cards in case board |
| `rumorIcon` | `assets/ui/rumor-card.png` | Rumor cards + rumor trail rows |
| `incidentIcon` | `assets/ui/incident-alert.png` | Incident / alert affordances |
| `lenoOverlay` | `assets/ui/leno-overlay.png` | Leno panel backdrop |

Site **`CaseBoardPanel`** and **`RumorTrailPanel`** resolve these with `assetUrl()` from `src/lib/play-api.ts`. Hotspot `icon` fields in the content pack are separate (per-hotspot overlays on location scenes). Static play HTML renders case board text without icons unless `gameShell.assets` is present in live state.
