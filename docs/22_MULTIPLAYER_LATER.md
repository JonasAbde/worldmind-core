# Multiplayer Later

## Recommendation

Start single-player. Design persistence so multiplayer can be added later.

## Later modes

- shared worlds.
- co-op districts.
- private servers.
- public creator worlds.
- asynchronous world sharing.

## Biggest risks

- simulation sync.
- griefing.
- AI cost explosion.
- content moderation.
- timeline conflicts.

## Design note

Multiplayer should not be added before event sourcing + permission system + content rules are mature.
